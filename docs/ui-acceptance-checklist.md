# Stret POS UI 验收清单

## 1) 验收设备矩阵

- 手机竖屏: 390 x 844
- 平板竖屏: 768 x 1024
- 笔记本: 1366 x 768

## 2) 全站通用验收项

- 页面无横向溢出，`body` 不出现横向滚动条。
- 主按钮高度一致，触控区域不少于 44px（手机）。
- 卡片圆角、阴影、边框风格一致（已接入 `ui-card` / `ui-panel` 的页面）。
- 文本层级清晰：标题、说明、状态标签不重叠不截断。
- 表格页面在小屏下可横向滑动查看，不遮挡操作按钮。
- 弹窗在小屏可完整操作，底部按钮可见且可点击。

## 3) 页面级验收项

### 登录与壳层

- `App` 登录页卡片在手机居中显示，角色按钮不换行错位。
- 顶部栏在手机显示菜单按钮和标题，不与语言选择器重叠。
- 抽屉菜单宽度适配手机，菜单项可滚动可点击。

### Dashboard

- 顶部统计卡在手机为 1 列，平板为 2 列，桌面为 4 列。
- 图表区域在手机不溢出，时间切换按钮可点击。
- 活动面板和底部合规模块在小屏不压缩变形。

### FinanceBoard

- 顶部利润卡金额在手机可换行显示，不超出容器。
- 两个导出按钮在手机仍可完整点击。
- VNPF/VAT 两块在小屏下间距、标题和金额对齐。

### WarehouseManagement

- 主表格在手机可横向滑动，列头和行内容对齐。
- 右侧 Feed 卡和 Ops 卡在手机上下排列，不重叠。
- “Load More Master Data” 按钮可完整显示且可点击。

### DriverApp

- 任务卡中 Pickup/Dropoff 两列在手机切为单列。
- “Accept Job” 和任务状态推进按钮在手机无遮挡。
- 进度条节点与标签在手机不重叠。

### RetailPOS

- 搜索框、扫码按钮、商品卡网格在手机可正常点击。
- 购物车列表和结算金额区域在小屏不溢出。
- 支付方式区域在手机按 2 列排布，字段不重叠。
- 收据弹窗在手机可滚动查看并完成打印按钮点击。

### SupplierManagement

- 顶部搜索与“New Supplier”按钮在手机纵向排布。
- 表格在小屏可横向滑动。
- 右侧统计卡在手机变为下方区块，信息完整。

### DigitalArchive

- 扫描按钮和批量导入按钮在手机纵向排布。
- 左侧分类卡、右侧文档卡在手机上下排列。
- 文档卡底部操作按钮可点击，状态标签不换行溢出。

### GlobalWholesaleLogistics

- 航运卡片在手机为纵向布局，检查按钮不被挤压。
- 审计按钮和提交按钮在小屏不重叠。
- 落地成本卡金额和提示块在手机可完整阅读。

### DigitalCustomsDocs

- 文档卡网格在手机单列，平板双列。
- 右侧预览卡 sticky 行为不影响手机滚动。
- “Download PDF / Print Physical” 在手机可点击且不出框。

## 4) 回归检查

- 刷新后登录状态仍保留（按角色）。
- 刷新后页面仍回到该角色上次访问视图。
- 左侧菜单仅显示当前登录角色菜单。
- `npm run build` 必须通过。
